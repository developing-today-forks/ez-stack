# ez-stack v0.1.5 Agent UX Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add 10 agent/human UX improvements to ez-stack: `--autostash`, `--json` output, non-interactive checkout, `--from` on create, `--if-changed` on commit, `$EDITOR` in pr-edit, `ez draft`/`ez ready`, `ez pr-link`, structured exit codes, SKILL.md, and README agent section.

**Architecture:** Features land in three layers: (1) low-level helpers in `git.rs` and `github.rs`, (2) CLI wiring in `cli.rs` + `main.rs`, (3) independent command implementations. All can be worked on after the CLI layer is established. Tests live in `#[cfg(test)]` modules in the same file as the code being tested.

**Tech Stack:** Rust edition 2024, rust-version 1.85, clap 4 (derive), serde_json, anyhow/thiserror. Git and GitHub operations shell out via `std::process::Command`. No new crate dependencies.

**Spec:** `docs/superpowers/specs/2026-03-11-v0.1.5-agent-ux.md`

fn parse_porcelain_dirty(output: &str) -> bool {

pub fn has_uncommitted_changes() -> Result<bool> {

pub fn stash_push() -> Result<bool> {
    if !has_uncommitted_changes()? {
        return Ok(false);
    }
    run_git(&["stash", "push", "--include-untracked", "-m", "ez-autostash"])?;
    Ok(true)
}

pub fn stash_pop() -> Result<()> {
    run_git(&["stash", "pop"])?;
    Ok(())
}

### Task 3: `ez sync --autostash`

**Files:**
- Modify: `src/cmd/sync.rs`

**Context:**
- `sync.rs::run` signature is currently `run(dry_run: bool)` — change to `run(dry_run: bool, autostash: bool)`
- The sync function does a lot: fetch, checkout trunk, fast-forward, delete merged branches, restack
- `git::stash_push()` returns `Ok(false)` if nothing to stash (no-op), `Ok(true)` if stashed
- We must call `stash_pop()` on ALL exit paths when we stashed — including early returns and `?` propagations
- Pattern: stash at top, wrap the rest in a closure or use a manual cleanup before each return/bail

#### Steps

- [ ] **Step 3.1: Update `sync.rs` function signature and add autostash logic**

At the top of `run`, after the dry_run block, add the autostash handling. The cleanest pattern in Rust without RAII is to track whether we stashed and ensure pop before every return:

```rust
pub fn run(dry_run: bool, autostash: bool) -> Result<()> {
    // ... existing dry_run block unchanged ...

    let stashed = if autostash {
        let did_stash = git::stash_push()?;
        if did_stash {
            ui::info("Stashed uncommitted changes (--autostash)");
        }
        did_stash
    } else {
        false
    };

    // Wrap the main body so we can always pop the stash on exit.
    let result = run_inner(stashed);

    if stashed {
        if let Err(e) = git::stash_pop() {
            ui::warn(&format!("Failed to pop autostash: {e}"));
        } else {
            ui::info("Restored stashed changes");
        }
    }

    result
}
```

Then extract the existing sync body (everything after the dry_run block) into a private `run_inner(_stashed: bool) -> Result<()>` function. The `_stashed` parameter is unused but keeps the signature clear.

- [ ] **Step 3.2: Extract the sync body into `run_inner`**

Cut the existing sync body (fetch → fast-forward → clean merged → restack → checkout → save → summary) and paste it as:

```rust

    // ... entire existing body of run() after the dry_run block ...
}
```

pub fn run(name: &str, message: Option<&str>, all: bool, from: Option<&str>) -> Result<()> {
    let mut state = StackState::load()?;
    let current = git::current_branch()?;

    // Determine the parent branch for the new branch.
    let parent = if let Some(base) = from {
        if !state.is_trunk(base) && !state.is_managed(base) {
            bail!(EzError::UserMessage(format!(
                "branch `{base}` is not tracked by ez — use trunk or a managed branch with --from"
            )));
        }
        base.to_string()
    } else {
        if !state.is_trunk(&current) && !state.is_managed(&current) {
            bail!(EzError::UserMessage(format!(
                "current branch `{current}` is not tracked by ez — switch to a managed branch or trunk first"
            )));
        }
        current.clone()
    };

    if git::branch_exists(name) {
        ui::hint(&format!("Use `ez checkout {name}` to switch to it"));
        ui::hint(&format!("Use `ez delete {name}` to delete and recreate it"));
        bail!(EzError::BranchAlreadyExists(name.to_string()));
    }

    // If a commit message was provided (only allowed without --from), commit first.
    if let Some(msg) = message {
        // message + from is blocked by clap conflicts_with, so parent == current here.
        if all {
            git::add_all()?;
        }
        if !git::has_staged_changes()? {
            ui::hint("Stage your changes first:  git add <files>");
            ui::hint(&format!("Or create the branch without committing:  ez create {name}"));
            bail!(EzError::NothingToCommit);
        }
        git::commit(msg)?;
        ui::info(&format!("Committed on `{current}`: {msg}"));
    }

    let parent_head = git::rev_parse(&parent)?;

    if from.is_some() {
        // Create at the tip of --from without switching branches.
        git::create_branch_at(name, &parent_head)?;
        ui::info(&format!("Branch `{name}` created from `{parent}` (not checked out)"));
        ui::hint(&format!("Run `ez checkout {name}` to switch to it"));
    } else {
        // Original behavior: create and switch.
        git::create_branch(name)?;
    }

    state.add_branch(name, &parent, &parent_head);
    state.save()?;

    if from.is_none() {
        ui::success(&format!("Created branch `{name}` on top of `{parent}`"));
    }
    Ok(())
}

## Notes for Implementers

1. **All commands output to stderr except `ez pr-link` and `--json` variants.** `println!` = stdout, `eprintln!` = stderr. Don't mix these up.

2. **`cargo fmt --all` is mandatory before every commit.** The CI checks this and will fail. Do it even if you think the code is already formatted.

3. **`cargo clippy -- -D warnings` must be clean.** Treat every warning as an error.

4. **Test location:** Tests go in `#[cfg(test)] mod tests { ... }` at the bottom of the file they test. No separate test files needed for unit tests.

5. **`todo!()` stubs should not be committed** except in Task 2 where they're placeholders for later tasks. All stubs must be replaced before Task 16.

6. **Parallelization:** Tasks 3–11 (Chunk 2) are fully independent and can be worked on by separate subagents simultaneously after Chunk 1 is committed. Tasks 13–15 (Chunk 4 docs) are also independent and can run in parallel with Chunk 2.
